Vue知识点进阶

Vue是什么

构建用户的渐进式框架，只关注视图层

渐进式： Vue为我们提供了核心的源码和周边的工具，如果只想做个数据渲染，那么只需要用到其 声明式渲染这个核心功能就可以了，然后如果项目界面分N多个小模块，就再用上 组件系统，再想做个单页应用或者移动端，这时候再用上 客户端路由,组件过多需要管理的时候再用上 大规模状态管理,当团队里面需要共同环境，在用上 构建工具
总结来说：当你用到哪个功能就去用他，不必一下子把全部东西都用上，可以一点一点往上去使用

Vue的两个核心点：

  1. 响应的数据绑定
    当数据发生改变时 —>自动更新视图
    利用Object.definedProperty中的setter/getter代理数据，监控数据的操作
  2. 可组合的视图组件

    把ui界面映射为组件树
    划分的组件可维护，可重用，可测试

虚拟DOM

由于js运行速度很快,大量操作DOM会导致js变慢。
在改变数据时重新渲染页面会把数据没有改变的DOM也重新渲染一遍造成资源的浪费
为了解决这个问题就在内存中生成与DOM相对应的数据结构,称之为 虚拟DOM 这样在页面更新时能智能计算出页面渲染所需的最小代价并应用到操作上

单向数据流

数据从父组件传递到子组件,只能单向绑定。
在子组件内部不应该修改父组件传递过来的值
改变prop的情况：

props验证

组件可以为props指定验证要求,如果未按指定验证要求，vue会发出警告
props:{
propA:Number,
propB:[String,Number]//多种类型
propC:{type:String,required:true}//必传,且为字符串
propD:{type:Number,default:100} //字符串类型,默认值为100
propE:{type:Number,default:function(){return 100}}
propF:{validator:function(value){return value >10}}//自定义验证规则
}
验证类型为原生构造器：String,Number,FunctionObject,Boolean,Array
